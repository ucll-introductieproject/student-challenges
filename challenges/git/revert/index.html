Git revert

Git revert kan soms handig zijn. Wanneer je een commit gepushed hebt die je ongedaan wil maken. Het git revert commando creÃ«ert een nieuwe commit die de laatste terug ongedaan maakt. Zo blijf je wel de historiek zien van alles wat er gebeurt is op de repository.

  • Maak een nieuwe folder "Challenge_02", plaats hierin een bestand "solution.txt" met de text "Challenge 2 completed.". Add, commit en push je wijzigingen.

  • Wijzig nu het bestand "solution.txt" met de tekst "Challenge 2 not completed". Add, commit en push je wijzigingen. Ga kijken in de browser dat de tekst van jou bestand effectief "not completed is".

  • Ga in de browser nu naar de url https://github.com/ucll-introductieproject/introductieproject-challenges-ucll-<plak_jouw_naam_hier>/commits/master. Je krijgt nu een overzicht van alle commits in jouw repository. De laatste commit willen we ongedaan maken, reverten. Hiervoor hebben we de commit ID nodig. Deze ID bestaat uit 6 karakters en kan je helemaal rechts van de lijn vinden. Klik op deze ID, je krijgt dan een vergelijking met wat er allemaal is aangepast in die commit. Zorg dat de ID op je klembord staat (je vindt die ook terug op deze pagina).

  • In git bash voer je volgend commando uit:

$ git revert <commit_ID>

  • Als je op enter duwt zal een text editor openen waarin je een boodschap moet typen. Wanneer je dat bestand sluit wordt de revert doorgevoerd. Let wel op, dit gebeurt weer enkel lokaal. Zorg ervoor dat dit ook op de remote repository gebeurt.

  • Kijk je solution.txt na in de browser, hierin zou terug "Challenge 2 completed" moeten staan.
